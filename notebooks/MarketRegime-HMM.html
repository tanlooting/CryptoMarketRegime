Market Regime Classification on Ethereum with Hidden Markov Model¶

Brief summary: In this study, we attempt to study the regime heteroskedasticity on the returns of Ethereum.

Financial time series tend to demonstrate volatility clusterings, large returns typically followed by small returns and vice versa. The heteroskedasticity of variance and the non-normal leptokurtic distribution of financial asset returns are also observed. Similarly, bitcoin was also found to exhibit regime heteroskedasticity.

Given that Ethereum is correlated to bitcoin most of the time, we would like to study the sudden changes in the price dynamics of ethereum through identifying its market regime, and potentially trade with different strategies during different market regimes.

Note that current work is still in progress
Further work:

  • try different features
  • sanity-check each regime
  • Incorporate a low-pass filter to identify optimal switching signal.
  • back-test strategies for each regime.

Reference: 1) Chappell, D. (2018) Regime heteroskedasticity in Bitcoin: A comparison of Markov switching models. Available at https://mpra.ub.uni-muenchen.de/90682/ 2) https://medium.com/@matthewwang_91639/algorithmic-factor-investing-with-market-regime-classification-6bc2f8c7168b

Hidden Markov Models¶

Hidden Markov Models is a model where a sequence of observable variables (in this case log returns), is generated by a sequence of internal hidden states m. The m states are not observed directly. The transition between each state i and j is based on the markov assumption: $ P(s_t = j | s_{t-1} = i) = p_{ij}(t) $

where $ p_{ij}(t) = p_{ij}, \forall t$

For an m-state model, we can construct an m x m transition matrix p(t).

In this exercise, we use HMMlearn python library to train the model

def plot_in_sample_hidden_states(hmm_model, df):
    """
    Plot the adjusted closing prices masked by 
    the in-sample hidden states as a mechanism
    to understand the market regimes.
    """
    # Create the correctly formatted plot
    fig, axs = plt.subplots(
        hmm_model.n_components, figsize = (10,10)
        #sharex=True, sharey=True
    )
    colours = cm.rainbow(
        np.linspace(0, 1, hmm_model.n_components)
    )
    for i, (ax, colour) in enumerate(zip(axs, colours)):
        mask = predictions == i
        ax.plot_date(
            df['unix'][mask], 
            df['close'][mask], 
            ".", linestyle='none', 
            c=colour
        )
        ax.set_title("Hidden State #%s" % i)
    plt.show()

Goodness-of-fit test¶

To assess each m-state model, we use 3 information criteria to estimate the information lost by each model.

  • Bayesian Information Criterion (BIC): $ k ln(n) - 2 ln(\hat{L})$
  • Akaike Information Criterion (AIC): $ 2k - 2 ln(\hat{L})$
  • Hannan-Quinn Information Criterion (HQC): $ -2ln(\hat{L}) + 2kln(ln(n))$

def bic_test(likelihood_fn, k, X):
    """ Log term omitted from likelihood_fn(x) because score function from hmmlearn already 
    computes the LOG-probability under the model
    
    Reference:

    https://hmmlearn.readthedocs.io/en/latest/api.html#hmmlearn.base._BaseHMM.score
    """
    return np.log(len(X))*k - 2*likelihood_fn(X)

def aic_test(likelihood_fn, k, X):
    return 2*k - 2 *likelihood_fn(X)

def hqc_test(likelihood_fn, k, X):
    return -2*likelihood_fn(X) + 2 * k * np.log(np.log(len(X)))

bic = []
aic = []
hqc = []
m_states = range(2,8)
lowest_bic = np.infty
em_iterations = 100
for hidden_states in m_states:
    
    model = hmm.GaussianHMM(n_components = hidden_states, covariance_type = 'full', n_iter = em_iterations)
    model.fit(obs) # shape = (n_samples, n_features)
    n_features = model.n_features
    
    #free parameters = for_means + for_covars + for transmat + for_startprob
    #for means & for_covars = n_features * hidden_states
    free_parameters = 2 * (hidden_states*n_features) + hidden_states*(hidden_states -1) + (hidden_states -1)
    
    #model score computes the log probability under the model
    bic.append(bic_test(model.score, free_parameters, obs))
    aic.append(aic_test(model.score, free_parameters, obs))
    hqc.append(hqc_test(model.score, free_parameters, obs))

Based on the computed AIC, BIC and HQC scores, the model with 3 states appear to have better fit (the lower the better). AIC indicated that the model with 5 states has a better fit than 3-states model.
